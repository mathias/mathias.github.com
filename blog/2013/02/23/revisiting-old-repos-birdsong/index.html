Revisiting Old Repos: Birdsong

The things that we leave behind or neglect can often be very interesting. They can also be a big source of embarassment; I know that I felt that way about the numerous empty repos and unfinished projects on my Github profile. Repos can be left behind for many different reasons: I might have a good idea but no time to execute on it, or I might lose interest and find something new to work on, or perhaps I don’t have the tools or skills to tackle an ambitious idea right now. Ambitious projects usually can be broken down into smaller, achievable chunks. If we don’t have the skills now, maybe we will some day. Or maybe we can build a piece of it.

I was recently inspired by my coworker Joe Nelson and his 1up Project to try and ship a project that I had let fall idle. Bendyworks has Growth Days on Friday in which everyone can work on whatever project they’d like, in an effort to allow for personal growth and Bendyworks’ growth.

So, scrolling to the bottom of my Github, I found one of the first repos that I had created on Github way back in 2008. No code had been pushed to it – it simply had the instructions for how a new repo.

The repo, Birdsong, was intended to be a collection of scripts for pulling down your user data from the APIs of various web services I was using at the time. Notably, Twitter and Last.fm data seemed like interesting things to grab.
